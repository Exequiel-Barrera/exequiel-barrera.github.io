What DOM is?

DOM stands for Document Object Model, and it is a way to represent the structure of a web page in a tree-like format. It allows us to access and manipulate the elements of a web page using JavaScript. For example, if we consider a web page as a tree, the DOM would be like the branches and leaves of that tree, where each element is a node in the tree.

Arrays

Arrays in JavaScript are like lists that can hold multiple values in a single variable.

  • Arrays are ordered lists of values.
  • Each value has a numeric index starting at 0.
  • You typically use index numbers to access data.

Objects

Objects in JavaScript are like containers that hold related data and functions

  • Objects are collections of key-value pairs.
  • Each value is accessed by its property name (key), not by index.
  • Keys are usually strings (or symbols), not numbers.
Arrays Objects
-Arrays Objects Ordered by index (0, 1, 2, …) -Organized by keys (name-value pairs)
-Access with fruits[0] -Access with person.name or person["name"]
-Best for lists/collections -Best for entities with attributes
-Maintains order automatically -No guaranteed order (though modern JS preserves insertion order for string keys)

Functions

Functions in JavaScript are reusable blocks of code that perform a specific task.

  • You define a function once.
  • You can call (use) it many times.
  • Functions can take inputs (parameters) and return outputs (results).

Why functions are useful

  • Reusability – Write code once, use it multiple times. Instead of repeating the same code, you call the function.
  • Organization – Break a big program into smaller, logical pieces. Easier to read and understand.
  • Avoid Repetition (DRY Principle: Don’t Repeat Yourself) Makes your code shorter and easier to maintain.
  • Easier Debugging – If something goes wrong, you only fix the function in one place.
  • Abstraction – You don’t need to know how it works internally every time, just what it does.
